# 视频管道

> 视频流的解复用、解码、渲染

---

## 概述

视频管道负责将 Android 端编码的 H.264/H.265 视频流解码并渲染到屏幕。

## 管道阶段

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 网络接收 │───►│ 解复用   │───►│ 解码    │───►│ 渲染    │
│ (UDP/TCP)│    │ Demuxer │    │ Decoder │    │ OpenGL  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

---

## 1. 解复用 (Demuxer)

### TCP 模式

| 文件 | `core/demuxer/video.py` |
|------|-------------------------|

帧格式：
```
[类型:1字节] [PTS:8字节] [包大小:4字节] [负载数据]
```

### UDP 模式

| 文件 | `core/demuxer/udp_video.py` |
|------|------------------------------|

帧格式：
```
[UDP Header: 24字节] [FEC Header: 4字节(可选)] [负载数据]
```

### 解复用器工厂

| 文件 | `core/demuxer/factory.py` |
|------|---------------------------|

根据连接模式自动选择正确的解复用器。

---

## 2. 解码 (Decoder)

| 文件 | `core/decoder/video.py` |
|------|-------------------------|

### 解码方式

| 方式 | 优先级 | 说明 |
|------|--------|------|
| NVDEC 硬解 | 1 | NVIDIA GPU 加速 |
| QSV 硬解 | 2 | Intel Quick Sync |
| FFmpeg 软解 | 3 | CPU 解码 |

### 输出格式

| 格式 | 说明 |
|------|------|
| NV12 | GPU 渲染首选 |
| YUV420P | 软解默认 |

### 延迟缓冲

| 文件 | `core/decoder/delay_buffer.py` |
|------|--------------------------------|

用于帧同步和延迟控制。

---

## 3. 渲染 (Rendering)

### OpenGL 窗口

| 文件 | `core/player/video/opengl_window.py` |
|------|---------------------------------------|

- 使用 QOpenGLWindow
- 支持 NV12 直接渲染
- GPU 着色器 YUV→RGB 转换

### 视频窗口容器

| 文件 | `core/player/video/video_window.py` |
|------|--------------------------------------|

- 窗口管理
- 宽高比保持
- 全屏切换

## FEC 纠错

| 文件 | `core/demuxer/fec.py` |
|------|------------------------|

### XOR 前向纠错

```
原始帧:  [F1] [F2] [F3]
FEC 帧:  [F1^F2^F3]

丢失 F2 时: F2 = F1 ^ F3 ^ (F1^F2^F3)
```

详见 [FEC 文档](fec.md)。
