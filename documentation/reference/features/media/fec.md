# FEC 前向纠错

> XOR 前向纠错，应对网络丢包

---

## 概述

FEC (Forward Error Correction) 通过发送冗余数据来恢复丢失的包，无需重传。

## 原理

### XOR 纠错

```
原始数据:  D1  D2  D3
FEC 校验:  P1 = D1 ⊕ D2 ⊕ D3

如果 D2 丢失:
D2 = D1 ⊕ D3 ⊕ P1
```

### 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| K | 4 | 每 K 个数据帧生成 1 个 FEC 帧 |
| 冗余率 | 25% | 每 4 帧额外 1 帧 |

---

## 实现细节

### 服务端 (编码)

| 文件 | `udp/SimpleXorFecEncoder.java` |
|------|--------------------------------|

```java
// 每 K 个帧生成一个 FEC 帧
for (int i = 0; i < k; i++) {
    xorInto(buffer, frames[i]);
}
sendFecFrame(buffer);
```

### 客户端 (解码)

| 文件 | `core/demuxer/fec.py` |
|------|------------------------|

```python
# 检测丢失并恢复
if frame_lost and have_fec:
    recovered = xor_frames(other_frames, fec_frame)
```

---

## FEC 头格式

```
┌──────────┬──────────┐
│ fec_idx  │  fec_k   │
│ (2 bytes)│ (2 bytes)│
└──────────┴──────────┘

fec_idx: 当前帧在 FEC 组中的索引 (0-K)
fec_k:   FEC 组大小 (通常为 4)
```

---

## 丢包恢复流程

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端 FEC 解码                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  接收: [F1] [F2] [F4] [P1]    (F3 丢失)                     │
│                                                             │
│  检测: fec_idx 缺失 2                                       │
│                                                             │
│  恢复: F3 = F1 ⊕ F2 ⊕ F4 ⊕ P1                              │
│                                                             │
│  输出: [F1] [F2] [F3] [F4]                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 局限性

1. **单帧恢复**: 每组只能恢复 1 个丢失帧
2. **连续丢包**: 连续丢失多帧无法恢复
3. **开销**: 增加 25% 带宽消耗

---

## PLI 请求

当 FEC 无法恢复时，发送 PLI (Picture Loss Indication) 请求关键帧：

| 类型 | 值 | 说明 |
|------|-----|------|
| TYPE_PLI | 0xE1 | 请求 I 帧 |

详见 [FEC PLI 协议规范](../../../FEC_PLI_PROTOCOL_SPEC.md)。
