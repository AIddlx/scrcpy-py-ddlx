## Phase 2: 性能优化

**目标**: 降低延迟，提高稳定性

**时间**: 预计 1-2 周

### 2.1 多帧缓冲架构

**问题**: 单帧缓冲导致帧丢失

**方案**:
```python
class FrameBuffer:
    def __init__(self, max_size=3):
        self._queue = deque(maxlen=max_size)
        self._lock = Lock()

    def push(self, frame):
        with self._lock:
            if len(self._queue) >= self.max_size:
                self._queue.popleft()  # 丢弃最旧
            self._queue.append(frame)

    def pop(self):
        with self._lock:
            return self._queue.popleft() if self._queue else None
```

**影响文件**:
- `scrcpy_py_ddlx/core/decoder/delay_buffer.py`
- `scrcpy_py_ddlx/core/player/video/opengl_window.py`

### 2.2 生产消费协调

## Phase 3: 架构重构

**目标**: 解决 GIL 竞争根本问题

**时间**: 预计 2-4 周

### 3.1 多进程架构

**当前问题**:
- 多进程解码器 UV 数据异常（已弃用）

**新方案**:
```
进程1 (网络): UDP 接收 + 分片重组 + FEC
进程2 (解码): 视频解码 + 音频解码
进程3 (渲染): OpenGL 渲染 + GUI
```

**挑战**:
- 进程间通信效率
- 共享内存同步
- 错误传播处理

### 3.2 共享内存优化

**当前问题**:
- SHM 读写竞争
- 数据格式不一致

**方案**:
- 环形缓冲区设计
- 无锁队列
- 内存对齐优化

### 3.3 Cython nogil 优化

**范围**:
- 颜色空间转换
- 数据复制操作
- 复杂计算

**示例**:
```cython
cdef void yuv_to_rgb_nogil(...) nogil:
    # C 代码，不持有 GIL
    ...
```

### 3.4 Python 3.12+ 评估

**特性**: Per-Interpreter GIL

**评估点**:
- 兼容性
- 性能提升
- 迁移成本
