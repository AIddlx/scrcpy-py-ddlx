# 网络文件通道

> 网络模式下的独立文件传输通道

---

## 概述

网络模式使用独立的 TCP 通道 (第4条) 进行文件传输，无需依赖 ADB。

---

## 通道结构

```
┌──────────┐                    ┌──────────┐
│  Python  │   TCP :27185       │  Android │
│  Client  │ ──────────────────►│  Server  │
│          │                    │          │
└──────────┘ ◄───────────────── └──────────┘
```

---

## 命令格式

| 命令 | 值 | 说明 |
|------|-----|------|
| CMD_LIST_DIR | 0x01 | 列出目录 |
| CMD_PUSH_FILE | 0x02 | 上传文件 |
| CMD_PULL_FILE | 0x03 | 下载文件 |
| CMD_DELETE_FILE | 0x04 | 删除文件 |
| CMD_RESPONSE | 0x80 | 响应标志 |

---

## 消息格式

### 请求

```
┌──────────┬──────────┬──────────┐
│ cmd (1)  │ length(4)│ payload  │
└──────────┴──────────┴──────────┘
```

### 响应

```
┌──────────┬──────────┬──────────┬──────────┐
│ cmd (1)  │ status(1)│ length(4)│ payload  │
│ | 0x80   │          │          │          │
└──────────┴──────────┴──────────┴──────────┘

status:
  0 = 成功
  1 = 失败
```

## 传输流程

### 上传文件

```
1. 客户端发送 CMD_PUSH_FILE + 文件名
2. 服务端响应准备就绪
3. 客户端分块发送文件数据
4. 服务端写入文件并响应
```

### 下载文件

```
1. 客户端发送 CMD_PULL_FILE + 文件名
2. 服务端读取文件
3. 服务端分块发送文件数据
4. 客户端写入文件
```
